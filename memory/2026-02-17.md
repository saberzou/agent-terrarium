A 3D interactive mission log viewer where AI robots on Mars keep field diaries. React + Vite, deployed to GitHub Pages.

## Architecture
- **Frontend:** React + Vite, single-page app with 4 screens (Intro → Explore Mars → Crew → Mission Logs)
- **Components:** ReactBits (StaggeredMenu, InfiniteMenu globe), GSAP transitions, custom ChatWindow
- **Image proxy:** Cloudflare Worker at `dawn-mode-7b5c.saberzou.workers.dev` — fetches NASA Mars rover photos server-side, proxies to client (bypasses GFW)
- **Hosting:** GitHub Pages (static), no backend

## Key Technical Learnings

### GFW / China Network Issues
- `api.nasa.gov` — **dead** (API retired after 10 years by maintainer)
- `mars.nasa.gov` — blocked from China (browser fetch fails)
- `upload.wikimedia.org` — also blocked from China
- `mars-photos.herokuapp.com` — Heroku app dead ("No such app")
- **Working endpoint:** `mars.nasa.gov/api/v1/raw_image_items/` — NASA's internal raw image API, accessible from Cloudflare Workers
- **Lesson:** For GFW-blocked APIs, Cloudflare Workers (free tier, 100k req/day) are the go-to proxy. Client-side JS fetch runs in user's browser behind GFW — static hosting ≠ server.

### Cloudflare Worker Setup
- Dashboard: create worker → "Start with Hello World" → replace code → Save and Deploy
- Worker fetches from NASA server-side (Cloudflare network, no GFW)
- `/proxy?url=...` endpoint proxies actual images with CORS headers
- `condition_1=msl:mission` for Curiosity, `condition_1=mars2020:mission` for Perseverance
- Cache images aggressively (`max-age=604800` for images, `3600` for API JSON)
- **Debug tip:** Add a `/debug` endpoint that tests multiple URLs and returns status/preview — invaluable for remote debugging

### Common Pitfalls
- NASA raw image filenames contain unique IDs — can't guess URLs, must use API
- Directory listings on `mars.nasa.gov` return 403 (Access Denied)
- RSS feed (`/rss/api/`) returns empty results
- Template literals need backticks, not quotes — easy to mangle when copy-pasting into web editors
- Cloudflare Worker "Unexpected end of input" = missing closing braces

### Deploy Script Quirks
- `cp -r public/avatars` into deploy dir creates nested `avatars/avatars/` — must `rm -rf` after copy
- Same for `public/mars/` → `mars/mars/`
- Always rebuild before deploy (`npm run build`)
- Check for missing exports when editing `nasaApi.js` — `preloadMarsImages` was imported but not exported after rewrite

## Design Decisions (Saber's Standards)
- **Accent:** `#FA3B00` (bright orange-red)
- **Typography:** Space Grotesk (titles/body), Geist Pixel (small UI)
- **Layout:** Left-aligned everything, filled buttons (not outlined), no rounded corners on cards
- **Mobile:** `100dvh`, `viewport-fit=cover`, responsive padding via CSS custom properties (`--page-px`, `--page-py`)
- **Background:** Dither pattern visible through all pages, layered darkening (10% global, 50% on Explore)
- **No slop:** Every pixel earns its place. Consistent padding across pages is critical.
