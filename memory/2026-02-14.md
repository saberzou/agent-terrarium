## Axel/Astra Bug — Root Cause Found
- Error: "Session file path must be within sessions directory"
- **Root cause**: OpenClaw bug in extensionAPI.js ~line 38390. `resolveSessionFilePath(sessionIdFinal, sessionEntry)` called WITHOUT `agentId`, defaulting to main agent's sessions dir. When sessionEntry.sessionFile has an absolute path to axel's sessions dir, path validation fails (`../../axel/sessions/...` starts with `..`).
- `agentDir` config is irrelevant — sessions dir is always derived from stateDir + agentId, not agentDir
